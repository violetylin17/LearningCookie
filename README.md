# LearningCookie
### 🧭 第 1 週：明確目標與建立 GitHub 專案
- **定義 AI agent 的用途**：翻譯AI(中文<->英文)。
- **選擇技術堆疊**：Python + Flask 是快速原型的好選擇。
- **建立 GitHub 專案**：
  - 設定主分支與開發分支。
  - 撰寫 README，說明目標、技術架構與使用方式。
  - 加入 issue 模板與 project board，方便協作追蹤。

---

### 🧱 第 2 週：開發核心邏輯
- **模型整合**：
  - 使用 OpenAI API、Hugging Face 或本地模型（如 llama.cpp）。
- **提示工程（Prompt Engineering）**：
  - 設計可重用的提示模板。
- **輸入/輸出處理**：
  - 可選 CLI、網頁表單或 Discord bot 等互動方式。

---

### 🔄 第 3 週：加入協作與測試機制
- **設定 GitHub Actions**：自動執行 lint、測試與格式化。
- **撰寫單元測試**：確保邏輯穩定。
- **啟用 issue 追蹤**：回報 bug 與提出功能需求。
- **加入基本日誌與錯誤處理**。

---

### 🧪 第 4 週：優化使用體驗與本地部署
- **加入簡易前端**：HTML 表單、Streamlit 或 CLI 包裝。
- **使用真實輸入測試**：涵蓋邊界情境。
- **撰寫使用說明文件**：README、Wiki 或教學影片。
- **選配 Docker 化**：方便分享與部署。

---

### 🚀 第 5 週：公開展示與迭代優化
- **製作 demo 影片或動圖**：展示成果。
- **撰寫開發紀錄或部落格**：分享學習歷程。
- **邀請回饋**：可在 Reddit、Hacker News 等平台發表。
- **規劃下一階段功能**：如多 agent 流程、記憶功能或網頁爬蟲。
